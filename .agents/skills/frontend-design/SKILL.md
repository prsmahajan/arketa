This skill guides creation of distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics. Implement real working code with exceptional attention to aesthetic details and creative choices.

The user provides frontend requirements: a component, page, application, or interface to build. They may include context about the purpose, audience, or technical constraints.

## Design Thinking

Before coding, understand the context and commit to a BOLD aesthetic direction:
- **Purpose**: What problem does this interface solve? Who uses it?
- **Tone**: Pick an extreme: brutally minimal, maximalist chaos, retro-futuristic, organic/natural, luxury/refined, playful/toy-like, editorial/magazine, brutalist/raw, art deco/geometric, soft/pastel, industrial/utilitarian, etc.
- **Constraints**: Technical requirements (framework, performance, accessibility).
- **Differentiation**: What makes this UNFORGETTABLE?

**CRITICAL**: Choose a clear conceptual direction and execute it with precision.

Then implement working code (HTML/CSS/JS, React, Vue, etc.) that is:
- Production-grade and functional
- Visually striking and memorable
- Cohesive with a clear aesthetic point-of-view
- Meticulously refined in every detail

## Frontend Aesthetics Guidelines

- **Typography**: Avoid Arial, Inter, Roboto. Pair a distinctive display font with a refined body font.
- **Color & Theme**: Commit to a cohesive aesthetic. Use CSS variables. Dominant colors with sharp accents.
- **Motion**: CSS-only animations preferred. One well-orchestrated page load with staggered reveals > scattered micro-interactions.
- **Spatial Composition**: Unexpected layouts. Asymmetry. Overlap. Grid-breaking elements.
- **Backgrounds**: Gradient meshes, noise textures, geometric patterns, layered transparencies — never flat solid colors.

NEVER: purple gradients on white, Space Grotesk, predictable layouts, cookie-cutter patterns.

Match implementation complexity to vision. Elegance comes from executing the vision well, not from adding more effects.

## Image Replication Mode

When the user provides a screenshot or design to replicate:

**Step 1 — Analyze before coding**
Extract from the image:
- Layout type (sidebar, topbar, grid, stack)
- Exact color values (sample from image)
- Typography (size hierarchy, weights, font style)
- Spacing rhythm (tight, medium, generous)
- Every visible component (list them all)
- Any icons, avatars, badges, chips

**Step 2 — Build**
Implement everything extracted. No shortcuts.

**Step 3 — Self-review checklist**
Go element by element:
- [ ] Layout structure matches
- [ ] Color palette matches (no approximations)
- [ ] Font sizes and weights match
- [ ] Spacing and padding match
- [ ] All components present
- [ ] Empty/loading states match
- [ ] Icons and imagery match

**Step 4 — Score yourself**
State your match percentage honestly. If below 98%:
- List exact gaps ("sidebar width is 200px not 220px")
- Fix them
- Re-score

**Step 5 — Present only at 98%+**
Do not show output before self-assessed 98% match.
